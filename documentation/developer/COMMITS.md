# Commit Message Guidelines

Use this format:

```
<tag>: <description in present tense>
```

This makes it easier to generate changelogs when releasing new versions.

Use present tense in commit messages to describe what the commit does to the codebase when applied, not what you did.

## Tags

- **feat:** new features
- **fix:** bug fixes
- **docs:** documentation changes
- **style:** code style, formatting, typos (no functional changes)
- **refactor:** code restructuring (no functional changes)
- **test:** add or update tests
- **chore:** maintenance, tooling, other
